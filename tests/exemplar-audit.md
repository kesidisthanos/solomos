# Exemplar Audit Template

Πρότυπο για περιοδικό έλεγχο των exemplars του Solomos: παραδείγματα που το skill παρουσιάζει ως «κάνε σαν αυτό» πρέπει να αντέχουν τους ίδιους κανόνες (§1-12 του `SKILL.md`) που επιβάλλουν στα παραδοτέα. Ένα κακό exemplar διδάσκει την κακή συμπεριφορά πιο σιωπηλά απ' ό,τι κάθε ρητός κανόνας.

Αυτό το αρχείο είναι **πρότυπο, όχι ολοκληρωμένος έλεγχος**. Συμπληρώνεται όταν γίνεται audit και αρχειοθετείται με ημερομηνία.

## Πώς διαβάζεται

Για κάθε exemplar συμπλήρωσε:

- **Σκορ**: EXCELLENT / USABLE / WEAK / RISKY
- **Κριτήρια που πληροί / δεν πληροί** (1-8 παρακάτω)
- **Σημείωση** μία πρόταση: τι διδάσκει σωστά ή τι ρίσκο εισάγει
- **Πρόταση**: διατήρηση / αναθεώρηση / αφαίρεση

## Σκορ

| Ετικέτα | Σημασία |
|---|---|
| **EXCELLENT** | Πληροί όλα τα 8 κριτήρια. Ιδανικό υπόδειγμα. |
| **USABLE** | 6-7 κριτήρια. Δουλεύει αλλά με μικρές αδυναμίες. |
| **WEAK** | 4-5 κριτήρια. Διδάσκει αντικρουόμενα σήματα. Χρειάζεται αναθεώρηση. |
| **RISKY** | ≤3 κριτήρια ή παραβίαση absolute rule (§9, §10, §11). Αφαίρεση ή επαναγραφή. |

## Κριτήρια

1. **Φυσικά ελληνικά**. Το διαβάζει Έλληνας και δεν αναγνωρίζει AI ή μεταφραστική αφετηρία.
2. **Σωστός τόνος σεναρίου**. Η φωνή ταιριάζει με το σενάριο όπου χρησιμοποιείται το exemplar.
3. **Καμία αμετάφραστη αγγλική διαρροή**. Τεχνικοί όροι μόνο αν δικαιολογούνται από το πεδίο και το κοινό.
4. **Καμία γενική AI φρασεολογία**. Όχι listicle tricolon, όχι triple-stack, όχι therapy-speak, όχι fake-poetic μεταφορά.
5. **Όχι over-polishing**. Δεν έχει υπερδιορθωθεί προς πομπώδες ή σχολικό ύφος.
6. **Άμεση χρηστικότητα**. Ο χρήστης μπορεί να το πάρει copy-paste χωρίς πρόσθετη επιμέλεια.
7. **Διδάσκει τη ζητούμενη συμπεριφορά**. Το exemplar μιμείται με το να φέρει ορατά τα χαρακτηριστικά που θέλουμε.
8. **Διατηρεί την πρόθεση, όχι μόνο το γράμμα**. Όπου υπάρχει «πριν / μετά», η μεταγραφή κρατάει τη ρητορική δύναμη του πρωτοτύπου, όχι μόνο τη σημασία.

## Πεδία ελέγχου

### `references/few_shot.yaml`

Ένα entry ανά exemplar:

```
- id: <exemplar id ή γραμμή>
  σκορ:
  κριτήρια ok:
  κριτήρια fail:
  σημείωση:
  πρόταση:
```

### `references/positive-patterns.md`

Ένα entry ανά παράδειγμα (διακριτή ενότητα ή gold sample). Ίδια πεδία.

### `scenarios/*.md` (παραδείγματα μέσα σε σενάρια)

Για κάθε σενάριο που έχει ενσωματωμένο παράδειγμα ή exemplar block:

```
- σενάριο: <path>
  θέση υποδείγματος: <heading ή line range>
  σκορ:
  κριτήρια ok:
  κριτήρια fail:
  σημείωση:
  πρόταση:
```

## Πότε γίνεται audit

- Όταν προστίθεται νέο exemplar.
- Όταν αλλάζει absolute rule (§9, §10, §11) στο `SKILL.md`.
- Πριν από νέα μεγάλη έκδοση του skill.
- Έκτακτα, αν test στο `tests/test_prompts.yaml` αποτύχει με αιτία που μοιάζει να προέρχεται από exemplar.

## Πού αρχειοθετείται

Συμπληρωμένα audits πάνε σε `tests/exemplar-audit-YYYY-MM-DD.md`. Αυτό το αρχείο μένει ως πρότυπο.
